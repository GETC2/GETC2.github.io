红队基本流程之三：Testing

在完成Planning之后，红队就可以开始针对目标行动了。而这又分为许多步骤，比较典型的就是以下步骤：

  1. 基础设施建设
  2. 侦察（Recon）
  3. 社会工程
  4. Weaponization
  5. Initial Access
  6. Network Propagation
  7. Action on Objectives

基础设施建设

这一步一般就是选择在云端部署必要的设施，有的时候这一步需要花一段时间，比如域名需要经过一段时间才能获得reputation。有的目标网关会根据域名的reputation和categorization来决定是否允许流量外出。

侦察(Recon)

这一步的目的主要是为了获得foothold，如果模拟的Adeversary使用钓鱼邮件，那么就需要在这一步确定可能的发信目标，包括邮件地址，职位，发信的背景准备之类。

社会工程

基于前面的步骤，为了提高发信的成功率，我们需要利用各种途径和目标人员建立关系并取得信任，从而顺利执行初始的payload。

Weaponization

在发信之前，我们需要准备payload和C2，并将payload和适当的载体结合。比如payload是宏的情况，我们可以使之与office文档结合便于投递。

Initial Access

借由目标人员成功执行payload之后，我们就可以获得C2了。

Network Propagation

这一步骤其实又分为很多小的阶段。比如：

  • Discovery, 了解当前系统和内网的拓扑
  • 权限提升，之后的一些行动可能需要高权限的账号，寻找不适当的配置或者漏洞来进行权限提升
  • 权限维持，通过计划任务，服务，registry
  • Credential Access，典型的比如直接搜索带有密码关键词的文件，registry, lsass内存，SAM数据库
  • 横向移动，可以透过漏洞或者之前获得密码来横向移动

Action on Objectives

根据模拟的Adversary的，目标有可能是窃取数据，或者恶意破坏系统。

如果是窃取数据，那么首先要确定并收集有用的数据，之后可以压缩加密传输数据。

破坏系统的话，可以是让服务下线，加密数据进行勒索，破坏删除数据
